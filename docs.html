Tracker Wygranych/Przegranych

/wl/{name}/{tag}/{region}
Śledź swoje dzienne wygrane i przegrane w trybie rankingowym. Zwraca prosty format pokazujący wyniki sesji.
GET /wl/{name}/{tag}/{region}

Parametry

  • ?resetTime=HHMM - Własny czas dziennego resetu (np. 0800)
  • ?sessionStart=timestamp - Własny początek sesji (w sekundach unix)

Przykład

Request: /wl/Szalony/123/eu?resetTime=0800
Response: 5W/2L

Zaawansowane W/L

/advanced_wl/{name}/{tag}/{region}
Rozszerzone śledzenie W/L z detalami ostatniego meczu, w tym zmiany RR i wynik.
GET /advanced_wl/{name}/{tag}/{region}

Parametry

  • ?resetTime=HHMM - Własny czas dziennego resetu (np. 0800)

Przykład

Response: 3W/1L (Ostatni: W +23RR)

Informacje o Randze (Rozszerzone)

/rank/{name}/{tag}/{region}
Pobierz aktualną rangę, RR i postęp do następnej rangi. Endpoint został połączony z funkcjonalnością dziennych statystyk, dzięki czemu możesz dołączyć do odpowiedzi bilans wygranych/przegranych, dzienną zmianę RR, zmianę RR z ostatniego meczu, a także statystyki (K/D/A) i agenta z ostatniej gry.
GET /rank/{name}/{tag}/{region}

Parametry

  • ?text="{rank} ({rr} RR) | Ostatnia gra: {lastStats} jako {lastAgent}"
  • Dostępne zmienne: {name}, {tag}, {lb}, {rank}, {rr}, {rrToGoal}, {goal}, {wl}, {dailyRR}, {lastRR}, {lastStats}, {lastAgent}
  • ?resetTime=HHMM - Własny czas dziennego resetu (np. 0800)
  • ?goalRank=RANK - Własny cel rangi (np. Immortal)

Przykład

Request: /rank/Szalony/123/eu?text={rank} ({rr}RR) | Ostatnio: {lastAgent} ({lastStats})
Response: Diament 2 (67RR) | Ostatnio: Jett (21/15/8)

Dzienne Statystyki

/daily/{name}/{tag}/{region}
Zwraca podsumowanie dzisiejszych statystyk gracza włącznie z rangą, bilansem W/L i zmianą RR.
GET /daily/{name}/{tag}/{region}

Parametry

  • ?resetTime=HHMM - Własny czas dziennego resetu (np. 0800)

Przykład

Request: /daily/Szalony/123/eu
Response: Diament 2 67RR | Bilans: 3W/1L | Dzisiaj: +33RR | Last: +23RR

Surowe Dane Rankingu

/rankraw/{name}/{tag}/{region}
Zwraca surowe dane API dotyczące gracza bez formatowania. Przydatne do debugowania i integracji.
GET /rankraw/{name}/{tag}/{region}

Przykład

Request: /rankraw/Szalony/123/eu
Response: { mmr: {...}, account: {...} }

Wyszukiwanie w Tabeli Wyników

/getrank/{position}
Wyszukaj graczy po ich pozycji w tabeli wyników (top 15000). Pokazuje rating, wygrane i link do trackera. Dane pochodzą ze statycznego pliku.
GET /getrank/{position}

Parametry

  • position: 1-15000 (pozycja w tabeli)

Przykład

Request: /getrank/50
Response: Player#TAG | Rating: 892RR | Wygrane: 156 | Tracker: ...

Nadchodzące Mecze

/nextmatch/{event}
Znajdź następny mecz dla określonego wydarzenia e-sportowego. Dane pobierane z VLR.gg.
GET /nextmatch/{event}

Parametry

  • event: Nazwa wydarzenia (np. VCT, EMEA)

Przykład

Request: /nextmatch/VCT
Response: Następny mecz na "VCT" to: Team A vs Team B za 2h 30m (2025-01-15 18:00)

Dzienne Mecze

/dailymatches/{event}
Pobierz wszystkie dzisiejsze mecze dla określonego wydarzenia e-sportowego. Dane pobierane z VLR.gg.
GET /dailymatches/{event}

Parametry

  • event: Nazwa wydarzenia (np. VCT, EMEA)

Przykład

Request: /dailymatches/VCT
Response: Dzisiejsze mecze VCT International: 14:00 Team A vs Team B | 17:00 Team C vs Team D

Strona ze Statystykami

/statystyki
Zwraca w pełni renderowaną stronę HTML z zaawansowanymi, dziennymi statystykami dla głównego gracza skonfigurowanego w aplikacji.
GET /statystyki

Przykład

Request: /statystyki
Response: Zwraca stronę HTML...

Health Check

/health
Zwraca status techniczny API. Służy do monitorowania działania serwisu. Zwraca kod 200, jeśli wszystko działa, lub 503, jeśli występuje problem.
GET /health

Przykład

Request: /health
Response: { "status": "operational", "timestamp": "...", ... }

Strona Statusu API

/status

GET /status

Przykład

Request: /status
Response: Zwraca stronę HTML z paskami statusu...
